Event & session creation
Trainer only

Trainers use this module to package their expertise into bookable events. Each event is a structured offering with a topic, pricing, and time slots that students can discover and book. This is the supply-side of the marketplace.

Step 2 — Session structure & pricing
Total sessionsHow many slots this event offers in total. E.g. 12 means 12 separate students can book this event independently. Trainer sets the cap.integer · required
Duration per sessionLength of each individual live session. Options: 30 min, 45 min, 60 min, 90 min. Determines how long the video room stays open.enum · required
Price per sessionAmount student pays in BDT for one session. Trainer earns 85% of this. Platform keeps 15%. Example: 1000 BDT → trainer gets 850 BDT.integer · required
Cancellation policyRules trainer sets: e.g. "Free cancel up to 24h before session", "No refund within 6h". Applied automatically at booking cancel.enum/text · required
Rescheduling policyWhether student can reschedule and how many hours before session is the cutoff. Trainer defines limit (e.g. "up to 1 reschedule, 12h notice").text · optional
Commission is auto-deducted. Trainer never needs to manually split — when session completes, 85% moves to trainer wallet and 15% to company wallet instantly.

Step 4 — Publish controls
Publish to marketplace Live
Event appears in public search. Students can find, view, and book it immediately. Trainer's profile shows the event. Booking notifications start arriving.
Save as draft Hidden
Event is saved but invisible to students. Trainer can edit freely. Useful for preparing an event in advance. No bookings possible in draft state.
Edit & cancel rules after publishing
Trainer can edit title, description, topics, and process overview at any time without affecting existing bookings.
Changing price or duration only affects new bookings — existing confirmed bookings honour the original terms.
Removing a published slot that already has a confirmed booking is not allowed. Trainer must cancel the booking first.
Cancelling the entire event is only allowed if zero bookings exist. Otherwise trainer must cancel each booking individually, triggering refunds per policy.

Business rules & constraints
Only trainers with admin-approved accounts can access the event creation form. Unapproved trainer accounts see a pending-approval screen instead.
A trainer can have multiple events live simultaneously across different categories.
Minimum price per session is 200 BDT. Platform enforces this to maintain quality signal.
Each event must have at least 1 available slot before it can be published.
Drafts are auto-saved every 30 seconds to prevent data loss.
Trainer receives in-app + email notification when the first booking is made on a newly published event.
Admin can unpublish or hide any event platform-wide if it violates guidelines, without trainer consent.
