User roles

Three distinct roles drive the platform. Each has a dedicated dashboard, specific permissions, and unique feature sets.

👨‍🎓

Student (User)

Self-registration
  • Browse & search events by category, type, date, price, trainer
  • Book individual session slots
  • Pay online via secure checkout
  • Join live video sessions
  • Rate and review trainers
  • View session history & transcripts
  • Receive booking reminders
  • Manage profile and preferences
🧑‍🏫

Trainer

Admin approval required
  • Create events with full session structure
  • Set price, topic, date/time, session count & duration
  • View and manage booked sessions
  • Start video sessions with students
  • Track wallet balance & withdrawals
  • Read student reviews and ratings
  • View earnings and session analytics
  • Manage availability calendar
🏛️

Admin / Company

Internal only
  • Approve or reject trainer registrations
  • Manage all users (students & trainers)
  • Monitor all sessions platform-wide
  • View total commissions collected
  • Generate revenue & activity reports
  • Handle disputes and refunds
  • Manage categories and interview types
  • Configure platform settings

Feature modules

Eight interconnected modules deliver the complete platform experience.

📋

Module 1 — Event & session creation (Trainer)

Trainer only
  • Create an event with: title, category, interview type, description, topics covered, process overview
  • Define session structure: total sessions (e.g. 12), duration per session (e.g. 30 min), price per session (e.g. 1000 BDT)
  • Set availability slots using a calendar picker; block unavailable dates
  • Publish event to the public marketplace or keep as draft
  • Edit or cancel events before any booking is made
🔍

Module 2 — Event discovery & browsing (Student)

Student only
  • Browse events by category (software, finance, product, etc.) and interview type (technical, behavioural, case study, HR)
  • Filter by price range, date, trainer rating, session duration, language
  • View full event detail page: trainer bio, topics, sample questions, reviews, session structure
  • Save events to wishlist / compare events side-by-side
  • See real-time availability of slots
📅

Module 3 — Booking & scheduling

Student
  • Select available slot from trainer's calendar; confirm time zone
  • Review booking summary (event, trainer, date, time, price) before payment
  • On successful payment, booking is confirmed and added to both parties' dashboards
  • Email + in-app notifications sent to student and trainer on booking
  • Reminder notifications: 24 hours before and 15 minutes before session
  • Rescheduling and cancellation policy managed per event rules
📹

Module 4 — Live video session

Both roles
  • Browser-based video interface — no app download required
  • Split-screen layout: student video on LEFT, trainer video on RIGHT
  • Session timer visible to both parties; auto-ends at scheduled duration
  • Controls: mute/unmute audio, enable/disable camera, end session
  • In-session text chat panel for sharing links, code, or notes
  • Screen sharing support for technical demonstrations
  • Session recording stored securely for 30 days (opt-in, both consent)
  • Automatic payment distribution triggers on session completion
💳

Module 5 — Payments

All roles
  • Student pays at booking (card, mobile banking, bKash)
  • Funds held in escrow until session completes
  • 15% auto-deducted to company wallet on completion
  • 85% credited to trainer wallet
  • Trainer withdraws to bank account on demand
  • Full payment history and receipts for all parties
⭐

Module 6 — Reviews & ratings

Student → Trainer
  • Student submits 1–5 star rating after session
  • Written review (optional) with character limit
  • Trainer can respond to reviews publicly
  • Aggregate rating displayed on trainer profile
  • Admin can moderate and remove abusive reviews
  • Reviews influence event ranking in search results
📊

Module 7 — Analytics

Trainer + Admin
  • Trainer: total earnings, sessions completed, upcoming count
  • Average rating trend over time
  • Revenue by month (bar chart)
  • Admin: platform-wide revenue, commission totals
  • Most active categories and trainers
  • Student retention rate and repeat bookings
🔔

Module 8 — Notifications

All roles
  • Email: booking confirmation, reminders, receipts
  • In-app: real-time bell notifications
  • SMS: session reminders (optional)
  • Push: mobile app notifications
  • Trainer notified on new booking, review, withdrawal
  • Admin notified on trainer applications, disputes

End-to-end workflow

The complete journey from event creation to post-session review, step by step.

Phase 1 — Setup

1
Trainer registers & gets approved

Trainer signs up, submits credentials and bio. Admin reviews and approves the trainer account.

2
Trainer creates an event

Sets title, category (e.g. "Software Engineering"), interview type, topics, process, session structure (12 × 30 min @ 1000 BDT each), and available time slots.

3
Event published to marketplace

Event appears in the public listing with all details, trainer profile, and real-time slot availability.

Phase 2 — Booking

4
Student discovers event

Searches or browses by category, reads event detail page, views trainer reviews, selects an available time slot.

5
Student pays securely

Checks out: 1000 BDT held in platform escrow. Slot is locked immediately.

6
Confirmation sent to both parties

Email + in-app notification. Session appears in both dashboards. Calendar invite generated.

Phase 3 — Live session (core experience)

7
Reminder sent 15 minutes before

Both student and trainer receive a "session starting soon" notification with a direct join link.

8
Video interface opens

Student's camera appears on the LEFT panel. Trainer's camera appears on the RIGHT panel. Session timer counts down. Chat sidebar available.

9
Session completes (auto or manual end)

Timer expires or trainer ends session. Recording saved (if opted in). Video call terminates cleanly.

Phase 4 — Payment & feedback

10
Automatic payment distribution

System auto-splits: 850 BDT → Trainer wallet | 150 BDT → Company wallet. Trainer receives receipt.

11
Student submits review

Prompted to rate (1–5 stars) and leave a written comment. Review published on trainer profile.

12
Trainer views feedback & balance

Dashboard shows new review, updated rating, and credited balance. Trainer can initiate withdrawal.

Every payment is automatically split at the moment a session is marked complete. No manual intervention needed.

Recommended tech stack

A production-ready stack suited for a Bangladesh-based SaaS platform with real-time video requirements.

Frontend

  • React.js — component-based UI for student and trainer dashboards
  • Next.js — SSR for SEO on public event pages
  • Tailwind CSS — rapid, responsive styling
  • WebRTC / Daily.co or Jitsi — browser-based video sessions
  • Socket.IO — real-time session status and chat
  • React Query — server state management

Backend

  • Node.js + Express or Django REST Framework
  • PostgreSQL — relational DB for all core data
  • Redis — session caching, real-time notifications
  • Celery / Bull — scheduled jobs (reminders, payouts)
  • JWT — stateless authentication
  • AWS S3 / Cloudflare R2 — session recording storage

Payments

  • SSLCommerz — Bangladesh's most widely used payment gateway
  • bKash API — mobile banking integration
  • Nagad API — secondary mobile wallet
  • Webhook-based payment confirmation
  • Escrow hold until session complete

Infrastructure & DevOps

  • Docker + Docker Compose — containerized dev and prod
  • AWS / DigitalOcean — cloud hosting
  • Nginx — reverse proxy and SSL termination
  • GitHub Actions — CI/CD pipeline
  • Sentry — error monitoring
  • Firebase Cloud Messaging — push notifications
